     - `updatedAt`
   - `tasks` table
     - `id` (primary key)
     - `title` (not null)
     - `description` (nullable)
     - `status` (not null, default: 'pending')
     - `dueDate` (nullable)
     - `userId` (foreign key referencing `users.id`)
     - `createdAt`
     - `updatedAt`

4. **API Contracts (REST)**:
   - **Authentication**:
     - `POST /api/auth/register`: Register a new user
     - `POST /api/auth/login`: Log in a user
     - `POST /api/auth/logout`: Log out a user
   - **Task Management**:
     - `GET /api/tasks`: Get all tasks for the authenticated user
     - `GET /api/tasks/:id`: Get a specific task by ID for the authenticated user
     - `POST /api/tasks`: Create a new task for the authenticated user
     - `PUT /api/tasks/:id`: Update an existing task by ID for the authenticated user
     - `DELETE /api/tasks/:id`: Delete a task by ID for the authenticated user

5. **Key Design Decisions**:
   - **Authentication**: The system will use JSON Web Tokens (JWT) for user authentication and authorization. Passwords will be hashed and salted using a secure algorithm like bcrypt.
   - **Task Management**: The tasks will be associated with the user who created them, ensuring data isolation and security. Task status will be represented as an enum (e.g., 'pending', 'in progress', 'completed', 'canceled').

   - **Task Management**:

7. **Scalability Considerations**:

   - **Monitoring and Alerting**: Set up comprehensive monitoring and alerting systems to quickly identify and address performance bottlenecks or system failures.

exports.authenticate = (req, res, next) => {
  const token = req.headers.authorization?.split(' ')[1];

  if (!token) {
    return res.status(401).json({ message: 'No token provided' });
  }

  try {
    const decoded = jwt.verify(token, JWT_SECRET);
    req.user = { id: decoded.userId };
    next();
  } catch (error) {
    return res.status(403).json({ message: 'Failed to authenticate token' });
  }
};
